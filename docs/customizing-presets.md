# Customising your presets

Set up the six preset buttons to play whatever TuneIn stations you
actually listen to.

## How presets work in this project

The speaker stores its six preset slots locally — the SoundTouch
firmware itself knows which station each button maps to. What it
*doesn't* know is the underlying stream URL; for that it asks the
on-speaker resolver. That's where this project comes in.

For each station you want as a preset, you need:

1. The station's **TuneIn ID** — a string like `s12345` or `s99999`.
2. A **resolver JSON file** at
   `/mnt/nv/resolver/bmx/tunein/v1/playback/station/<id>` containing the
   actual stream URL. Generated by `resolver/build.py`.
3. The station **assigned to a preset slot** (1–6) on the speaker.
   Either by holding the preset button on the speaker itself, or by
   POSTing to the speaker's local API.

## Step 1 — Find the TuneIn ID for your station

Open [tunein.com](https://tunein.com) in a browser, search for the
station, and look at the URL of the result. It'll be of the form:

```
https://tunein.com/radio/Some-Station-Name-s12345/
```

The trailing `s12345` is the ID. Some stations use a `t` prefix
(topics) or `p` prefix (podcasts) — for live radio you want the
`s` prefix.

If you've previously stored a station as a preset on the speaker (and
the speaker has been online recently enough to know about it), you can
also read the IDs out of the speaker:

```bash
curl http://<speaker-ip>:8090/presets
```

Look at each `<ContentItem>` element — `location="/v1/playback/station/sNNNNN"`
contains the ID, and `<itemName>` has the friendly name.

## Step 2 — Edit your station list

Copy the example and edit:

```bash
cd resolver/
cp stations.example.json stations.json
$EDITOR stations.json
```

`stations.json` is a JSON array of `[id, name]` pairs. Order matters
only for your own readability — the slot a station ends up in is
chosen in step 4, not by position in this file.

```json
[
  ["s12345",  "Your favourite station"],
  ["s23456",  "Another station"],
  ["s34567",  "A third"]
]
```

Six entries is the natural number (one per preset button) but you
can list more — the speaker only has six slots though.

`stations.json` is gitignored on purpose — it's your personal data,
not something to commit.

## Step 3 — Build the resolver files

```bash
python3 build.py
```

For each entry in `stations.json`, this calls TuneIn's public API
(`opml.radiotime.com/Tune.ashx`) with the parameters that unlock the
real partner-routed stream URLs (`formats=mp3,aac&lang=de-de` —
without these, some stations return a placeholder "not compatible"
URL), reshapes the response into the JSON shape the speaker expects,
and writes one file per station ID to the current directory.

If a station has no compatible streams (geo-restricted to a region
you're outside, or simply not available via that endpoint), you'll
see `WARN sNNNNN ... no compatible streams` and no file is written
for that station. Skip it or use a different station.

## Step 4 — Push to the speaker and assign to a slot

```bash
SPEAKER_IP=<your-speaker-ip>
SCP="scp -O -oHostKeyAlgorithms=+ssh-rsa -oUserKnownHostsFile=/dev/null -oStrictHostKeyChecking=no -oConnectTimeout=10"
$SCP s* root@$SPEAKER_IP:/mnt/nv/resolver/bmx/tunein/v1/playback/station/
```

Then assign one of your stations to a preset slot. Two ways:

### Option A — On the speaker itself

While the station is playing on the speaker (e.g. you got it playing
via Bluetooth from your phone, or from another preset that already
points at it), hold the desired preset button (1–6) for a few seconds.
This is how Bose meant it to work.

### Option B — Via the browser admin

Open `http://<speaker-ip>:8181/`, search or browse to the station,
open it, and tap one of the six preset slots in the assignment grid.
The admin handles the `/storePreset` POST and atomically rewrites the
matching `/v1/playback/station/<id>` resolver entry. This is the
day-to-day path once the admin is deployed.

### Option C — Via the speaker's local API

There's a wrapper script:

```bash
./scripts/store-preset.sh <speaker-ip> <slot> <station-id> "<station-name>"
# e.g.
./scripts/store-preset.sh 192.168.1.42 1 s12345 "My favourite station"
```

Or, equivalently, by hand. Note the body must wrap the `ContentItem` in
a `<preset id="N">` element — a bare `ContentItem` returns
`1019 CLIENT_XML_ERROR`:

```bash
SLOT=1                              # 1..6
ID=s12345                           # the TuneIn ID
NAME="Your favourite station"

curl -X POST -H 'Content-Type: application/xml' \
  -d "<preset id=\"$SLOT\"><ContentItem source=\"TUNEIN\" type=\"stationurl\" \
        location=\"/v1/playback/station/$ID\" sourceAccount=\"\" \
        isPresetable=\"true\"><itemName>$NAME</itemName></ContentItem></preset>" \
  "http://$SPEAKER_IP:8090/storePreset?id=$SLOT"
```

The speaker stores the preset; pressing the preset button on the
hardware now triggers a lookup at
`http://127.0.0.1:8181/v1/playback/station/$ID`, which our resolver
serves from disk.

## Verifying

Press the preset and check what the speaker thinks it's doing:

```bash
curl -X POST -H 'Content-Type: application/xml' \
  -d '<key state="press" sender="Gabbo">PRESET_1</key>' \
  http://$SPEAKER_IP:8090/key
curl -X POST -H 'Content-Type: application/xml' \
  -d '<key state="release" sender="Gabbo">PRESET_1</key>' \
  http://$SPEAKER_IP:8090/key
sleep 4
curl http://$SPEAKER_IP:8090/now_playing
```

You're looking for `source="TUNEIN"`, `playStatus="PLAY_STATE"`, and
the right station name.

## When a stream stops playing

TuneIn occasionally rotates the actual stream URL for a station —
re-run the refresh script:

```bash
./scripts/refresh-streams.sh <speaker-ip>
```

It re-runs `build.py`, then `scp`s the fresh files to the speaker. No
reboot needed.

## Adding a station that isn't on TuneIn

Out of scope for v1. The resolver expects TuneIn-shaped responses, and
the speaker's preset mechanism for non-TuneIn sources (Spotify, AUX,
Bluetooth) works through entirely different code paths that don't
involve this project. If you want a custom direct-URL station, you'd
need to:

1. Hand-edit a Bose-shaped JSON file with your direct stream URL.
2. Push it to `/mnt/nv/resolver/bmx/tunein/v1/playback/station/<some-id>`
   (give it any unique `s`-prefixed ID).
3. Assign as a preset via Option C above (the browser admin won't help
   for a non-TuneIn id — its assignment grid POSTs through the
   resolver's TuneIn-shaped path).

That works but isn't supported by `build.py`. Patches welcome (see
[../CONTRIBUTING.md](../CONTRIBUTING.md)).
